# MNIST Digit Classification with MLP

## Описание проекта

В данном проекте решается задача классификации рукописных цифр из датасета MNIST с использованием полносвязных нейронных сетей (MLP) в библиотеке PyTorch.

Цель работы — сравнить несколько архитектур нейронных сетей и определить, какая из них показывает наилучшее качество классификации.

В ходе экспериментов исследовалось влияние:
- количества скрытых слоёв
- Batch Normalization
- Dropout
- степени регуляризации

---

# Датасет

Используется датасет **MNIST**, содержащий изображения рукописных цифр.

Характеристики датасета:

- 70 000 изображений
- размер изображения: **28 × 28**
- 10 классов (цифры **0–9**)
- 60 000 изображений для обучения
- 10 000 изображений для тестирования

Датасет автоматически загружается через `torchvision.datasets.MNIST`.

# Архитектуры моделей

В рамках работы были реализованы и протестированы три архитектуры.

## Эксперимент 1 — Baseline MLP

Простая полносвязная сеть.

Архитектура:

```
Flatten
Linear (784 → 128)
ReLU
Linear (128 → 10)
```

Особенности:

- 1 скрытый слой
- без регуляризации
- используется как базовая модель для сравнения

---

## Эксперимент 2 — Regularized MLP

Более глубокая сеть с регуляризацией.

Архитектура:

```
Flatten
Linear
BatchNorm
ReLU
Dropout (0.5)

Linear
BatchNorm
ReLU
Dropout (0.5)

Linear → 10
```

Особенности:

- 3 скрытых слоя
- Batch Normalization
- Dropout = 0.5
- сильная регуляризация

---

## Эксперимент 3 — Compact Regularized MLP

Более компактная архитектура с умеренной регуляризацией.

Архитектура:

```
Flatten
Linear
BatchNorm
ReLU
Dropout (0.2)

Linear
BatchNorm
ReLU
Dropout (0.2)

Linear → 10
```

Особенности:

- 2 скрытых слоя
- BatchNorm
- Dropout = 0.2

---

# Обучение модели

Модели обучаются с использованием:

- оптимизатора **Adam**
- функции потерь **CrossEntropyLoss**

В процессе обучения строятся графики:

- training loss
- validation loss

Это позволяет оценить сходимость модели и возможное переобучение.

---

# Результаты экспериментов

| Эксперимент | Архитектура | Val Loss | Accuracy | Precision | Recall | F1-score |
|-------------|-------------|----------|----------|----------|----------|----------|
| exp_1_baseline | 1 скрытый слой | 0.0895 | 0.9745 | 0.9747 | 0.9741 | 0.9743 |
| exp_2_regularized_mlp | 3 слоя, BatchNorm, Dropout=0.5 | 0.0763 | 0.9786 | 0.9785 | 0.9786 | 0.9785 |
| exp_3_compact_mlp | 2 слоя, BatchNorm, Dropout=0.2 | **0.0684** | **0.9818** | **0.9818** | **0.9817** | **0.9817** |

---

# Выводы

Лучшие результаты показала модель **exp_3_compact_mlp**.

Причины:

- модель достаточно глубокая, чтобы распознавать различные особенности написания цифр
- используется Batch Normalization, стабилизирующий обучение
- умеренная регуляризация (Dropout = 0.2) предотвращает переобучение, но не мешает обучению

Более сложная модель из эксперимента 2 использует более сильный Dropout (0.5), что может затруднять обучение. В результате компактная архитектура из эксперимента 3 показывает лучший баланс между сложностью модели и регуляризацией.

Итоговая точность модели на тестовой выборке составила **98.18%**.
